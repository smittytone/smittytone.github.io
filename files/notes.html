Squinter 1.0.117 (11 November 2016)

Improvements/Additions

  • Update Squinter projects file type to 2.1:
    • Projects will now store paths to files relative to the project file location
    • Squinter will offer to update old projects to file type 2.1 (this is optional)
    • Preference for file path display added: absolute path, relative to project, and relative to user's home directory
    • When searching for mis-referenced libraries, Squinter now looks relative to the project file rather than agent or device code files
  • Improved file handling:
    • The 'add file(s) to project' code has been rewritten
    • The 'open project' code has been rewritten
    • Better handling of multiple projects with the same name
  • Better handling and checking of Electric Imp libraries
  • The display of device, model and project information has been improved
  • The display of bulk-loaded log messages has been improved
  • Preferences panel now tabbed to improve clarity

Bug Fixes

  • Correctly process import statements referencing sub-directories
  • Correctly allow the retrieval of logs from currently unassigned devices
  • Fix some issues encountered when opening very old project files

Bug Fixes

  • Display Upload button only when a project is linked to a model
